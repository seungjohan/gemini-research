Organizing Your Knowledge OS

Choosing how to "divide by subject" for both IDE navigation and Obsidian Graph View

A

Subject-Based Folders

Organize files into /wiki/career/, /wiki/stocks/, etc.

  • IDE: Folders are visible and collapsible.
  • Graph View: Folders can be colored differently (using "Groups").
  • Trade-off: Breaks the current "flat" mandate; requires updating automation scripts.
B

Enhanced Tagging + MOCs

Keep the directory flat, but use #career, #stocks tags and "Maps of Content" (MOCs).

  • IDE: Flat list, but easily filtered by file name prefix (e.g., career-xxx.md).
  • Graph View: Best for Obsidian. You can color nodes by tag and see relationships better.
  • Trade-off: Harder to "browse" by folder in a standard file explorer.
